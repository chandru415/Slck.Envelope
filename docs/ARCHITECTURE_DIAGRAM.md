## Key Integration Points

| Component | Responsibility | What It Adds |
|-----------|----------------|--------------|
| **RequestIdMiddleware** | Request correlation | X-Request-Id header, TraceIdentifier |
| **ApiEnvelopeMiddleware** | Error handling | Wraps exceptions in envelope |
| **MediatR** | Request routing | Decoupling, pipeline behaviors |
| **LoggingBehavior** | MediatR logging | Logs request start/end |
| **ValidationBehavior** | Input validation | FluentValidation, early exit |
| **ObservableCommand/Query** | Instrumentation | OTEL spans, Serilog scopes |
| **Handler.HandleAsync()** | Business logic | Your code with Logger/ActivitySource |
| **EnvelopeResult** | Response formatting | Consistent JSON envelope |

## Summary

When you combine MediatR with Slck.Envelope Observability:

? **MediatR handles**: Request routing, pipeline behaviors, validation  
? **ObservableCommand/Query handles**: OTEL tracing, Serilog logging  
? **Envelope handles**: Response format, error wrapping  
? **You write**: Just the `HandleAsync()` business logic  

Result: **Production-ready observable CQRS with minimal code!** ??
